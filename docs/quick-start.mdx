# Quick Start

This progressive documentation will explore all things related to the [**unified-doc**][unified-doc].

## [Readme](./readme)

An overview of the `unified-doc` initiative, covering motivations for unified document APIs and renderers.

## [Demos](./demos/ebook)

A small collection of document-based applications showcasing diverse features that are easily implemented using `unified-doc` packages.

> Demos are implemented using the [`unified-doc-react`](./packages/unified-doc-react) renderer, but the concepts are easily translatable to other renderers since the renderer packages are thin wrappers implementing the core [`unified-doc`](./packages/unified-doc) APIs.

## Packages

### Core
Core unified document APIs covering annotations, selecting text, reading/saving files, and many more to come.
- [`unified-doc`](./packages/unified-doc)

### Parsers
Parsers used in `unified-doc`.
- [`rehype-parse`](https://github.com/rehypejs/rehype/tree/master/packages/rehype-parse)
- [`remark-parse`](https://github.com/remarkjs/remark/tree/master/packages/remark-parse)
- [`unified-doc-parse-text`](./packages/unified-doc-parse-text)
- `unified-doc-parse-docx` (TODO)
- `unified-doc-parse-pdf` (TODO)

### Renderers
Renderers implementing `unified-doc` in vanilla JS or specific libraries.
- [`unified-doc-react`](./packages/unified-doc-react)
- `unified-doc-vue` (WIP)
- `unified-doc-html` (WIP)

### Utils
Hast utils used in `unified-doc`
- [`unified-doc-util-annotate`](./packages/unified-doc-util-annotate)
- `unified-doc-download` (WIP)
- `unified-doc-search` (WIP)
